# GlyphLoom 范围与不做清单

> 写给未来的自己与协作者：遇到“酷但跑题”的想法时，先翻这里。

## 我们重点做什么？

1. **静态批量本地化流水线**：提取 → 翻译 → QA → 导出，全部可配置、可编程。  
2. **LLM 驱动的本地化质量**：聚焦术语、一致性、占位符保护、风格控制等“LLM 擅长”的内容。  
3. **可扩展架构**：核心只做 pipeline 与抽象层，Ren'Py/Unity/Unreal 等引擎支持通过 adapter/miner 插件扩展。  
4. **透明的配置与接口**：配置文件优先，CLI/GUI/SDK 共用一套 API；明确区分 config 与 secrets。

## 我们明确不做什么？

1. **实时叠加机翻**：不做 LunaTranslator、Textractor 这类实时外挂。GlyphLoom 只服务“静态本地化”。  
2. **盗版补丁 / 破解分发**：不做资源篡改、密钥绕过等功能，只处理文本层工具链。  
3. **代管或薅第三方 API key**：用户自己管理 Key，我们只提供 `api_key_env` 这种引用方式。  
4. **非法绕限流或刷接口**：不做批量创建账号、伪装请求之类的“黑魔法”。  
5. **GUI 内部的重型编辑器**：Stage 7 规划的“内联文本编辑”只做轻量操作，不会变成全功能 DTP/IDE。  
6. **自动化部署/分发游戏**：只负责生成本地化结果/补丁；安装、打包、分发交给用户。

## 如果未来想调整？

- 可以新增条目，但务必在 PR / 评审中讨论，并记录为什么要破例。  
- 若确实需要探索新的产品方向，建议单列项目或插件，而不是在 core 中堆功能。  
- 按照 Roadmap 阶段优先推进，任何超出范围的需求，一律先记在 design_notes/todo 中，再决定是否纳入。
